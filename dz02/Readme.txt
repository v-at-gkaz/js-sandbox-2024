Организовать проект с несколькими файлами, содержащими переменные окружения (например, .env_prod, .env_dev)
и настроить скрипты в package.json проекта на запуск проекта в различных режимах (например, prod, dev).

Работа в различных режимах подраземевает подключение к различным базам данных и разный режим
отладки (см. project_06_env (DEBUG))

Подключение к СУБД обычно подразумевает несколько параметров:

- хост (ip адрес или доменное имя)
- порт
- логин
- пароль
- имя базы данных
- имя основной схемы базы данных (Postgres)

Программа при запуске должна выводить значение переменных окружения для различных режимов.
